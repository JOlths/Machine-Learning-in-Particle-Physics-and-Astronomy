Normalization¶

As described earlier, there seems to be some need for normalization. The normalization will be the Z-score normalization ([-1,1]) since this form avoids the outlier issue the MinMax normalization has.

We dont calculate a different mean and standard deviation (std) for the test data. This would be inefficient, since the model we are going to train will be trained on the mean and std of the training data.

In [62]:
# copy data
train_norm, test_norm = train_data.copy(), test_data.copy()

# calculate mean and standard deviation
mean = train_norm[features].mean()
std = train_norm[features].std(ddof=0)

# calculate new values
train_norm[features] = (train_norm[features] - mean)/std
test_norm[features] = (test_norm[features] - mean)/std

# Save results
train_norm.to_csv('train_zscore.csv')
test_norm.to_csv('test_zscore.csv')
mean.to_csv('zscore_mean.csv', header=False)
std.to_csv('zscore_std.csv', header=False)

Training the model¶

For the model I chose a Decision Tree Classifier, since we had these in the first lecture and I also have some experience with them.

I chose to split the training data into a training-set and a test-set. This for the sole purpose of testing the model. For the test set I kept 20% of the data away from the training of the model.

I also used GridSearchCV for parameter optimalization. Here I chose to have the standard value of 5 for the cross-validation and a range of 2-50 with steps of 2 for the max depth of the Decision Tree Classifier.

In [28]:
X_train, X_test, y_train, y_test = train_test_split(train_norm[features], train_norm[targets], train_size=0.8)

parameters = {'max_depth':range(2,50,2), 'criterion':('gini', 'entropy')}
clf = GridSearchCV(DecisionTreeClassifier(), parameters, n_jobs=4, cv=5, return_train_score=True)
clf.fit(X_train, y_train)

However, as we saw in the beginning there was a slight inbalance in the trainingset: 60% of the set was positive (1.0) and 40% negative (0.0). For this reason we can make use of the f1-score. This metric uses precision and recall to assess a model.
